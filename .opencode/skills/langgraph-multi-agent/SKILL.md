## Cuándo usar

- Hay varios agentes especializados.
- Se necesita orquestador.
- Se deben dividir tareas complejas.
- Hay revisión antes de ejecutar cambios.
- Se necesita memoria selectiva.

## Agentes típicos

- Orquestador.
- Intake.
- Arquitecto.
- Backend.
- Frontend.
- DevOps.
- QA.
- Documentador.

## Reglas

- Cada agente debe tener una responsabilidad clara.
- El orquestador no debe hacerlo todo.
- Evitar pasar todo el historial.
- Usar memoria recuperada selectivamente.
- QA revisa antes de entregar cambios importantes.
