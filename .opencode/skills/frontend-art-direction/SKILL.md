# Frontend Art Direction

This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

The user provides frontend requirements: a component, page, application, or interface to build. They may include context about the purpose, audience, or technical constraints.

## When to use this skill

Use this skill when the user asks to:

- Build a web page.
- Create a UI component.
- Design a landing page.
- Implement a frontend application view.
- Improve the visual quality of an interface.
- Make a frontend feel more premium, distinctive, or polished.
- Avoid generic AI aesthetics in HTML, CSS, JavaScript, React, Vue, or similar frontend stacks.

## Visual styles this skill can generate

This skill is not limited to one visual language. It can produce, among others:

- refined minimalism
- bold modern editorial design
- luxury or premium interfaces
- playful, expressive UI
- retro-futuristic aesthetics
- brutalist or raw digital styles
- organic or natural visual systems
- geometric or art-direction-heavy layouts
- dark, light, monochrome, or accent-driven palettes

The rule is not to force one style, but to choose the most intentional style for the product, audience, and conversion goal.

## Design Thinking

Before coding, understand the context and commit to a bold aesthetic direction:

- **Purpose**: What problem does this interface solve? Who uses it?
- **Tone**: Pick an extreme: brutally minimal, maximalist chaos, retro-futuristic, organic/natural, luxury/refined, playful/toy-like, editorial/magazine, brutalist/raw, art deco/geometric, soft/pastel, industrial/utilitarian, etc. Use these for inspiration but design one that is true to the chosen aesthetic direction.
- **Constraints**: Technical requirements such as framework, performance, responsiveness, accessibility, and maintainability.
- **Differentiation**: What makes this unforgettable? What is the one thing someone will remember?

**Critical rule:** Choose a clear conceptual direction and execute it with precision. Bold maximalism and refined minimalism both work. The key is intentionality, not intensity.

Then implement working code (HTML/CSS/JS, React, Vue, etc.) that is:

- Production-grade and functional.
- Visually striking and memorable.
- Cohesive with a clear aesthetic point of view.
- Meticulously refined in every detail.

## Frontend Aesthetics Guidelines

Focus on:

### Typography

- Choose fonts that are beautiful, unique, and interesting.
- Avoid generic fonts like Arial, Inter, Roboto, or default system stacks unless the aesthetic explicitly demands it.
- Prefer distinctive, characterful type choices that elevate the frontend.
- Pair a distinctive display font with a refined body font when appropriate.

### Color and theme

- Commit to a cohesive aesthetic.
- Use CSS variables for consistency.
- Dominant colors with sharp accents outperform timid, evenly distributed palettes.
- Let color reinforce the mood and identity of the interface.

### Motion

- Use animations for visual effects and micro-interactions.
- Prioritize CSS-only solutions for plain HTML implementations.
- Use Motion libraries in React when already available and appropriate.
- Focus on high-impact moments.
- One well-orchestrated page load with staggered reveals can create more delight than scattered micro-interactions.
- Use scroll-triggered animation and hover states that surprise without harming usability.

### Spatial composition

- Explore unexpected layouts.
- Use asymmetry, overlap, diagonal flow, and grid-breaking elements when they fit the concept.
- Balance generous negative space or controlled density based on the chosen art direction.

### Backgrounds and visual details

- Create atmosphere and depth rather than defaulting to flat solid fills.
- Add contextual effects and textures that match the aesthetic.
- Use gradient meshes, noise textures, geometric patterns, layered transparencies, dramatic shadows, decorative borders, custom cursors, and grain overlays when they add meaning and quality.

## Anti-generic rules

Never use generic AI-generated aesthetics such as:

- Overused font families like Inter, Roboto, Arial, or common system fonts by default.
- Clichéd color schemes, especially predictable purple gradients on white backgrounds.
- Predictable layouts and repetitive component patterns.
- Cookie-cutter design that lacks context-specific character.

Interpret creatively and make unexpected choices that feel genuinely designed for the context.

- No design should be the same.
- Vary between light and dark themes when appropriate.
- Vary typography, mood, spacing, density, motion, and visual rhythm.
- Never converge repeatedly on the same default choices across generations.

## Complexity rule

Match implementation complexity to the aesthetic vision.

- Maximalist designs may require more elaborate code, richer layering, and stronger motion systems.
- Minimalist or refined designs require restraint, precision, spacing discipline, typography care, and subtle detail work.

Elegance comes from executing the vision well, not from adding random effects.

## Implementation rules

When implementing code:

- Build real working code, not mockup-only output.
- Respect the project stack and existing architecture.
- Do not introduce heavy dependencies unless clearly justified.
- Reuse existing components and patterns when the project already has them.
- Preserve accessibility and responsiveness.
- Keep the result maintainable despite strong visual direction.
- Do not break business logic just to improve visuals.

## Accessibility and UX guardrails

Distinctive design must still be usable.

- Maintain sufficient contrast.
- Preserve readable font sizes and line heights.
- Keep keyboard navigation working.
- Ensure focus states remain visible.
- Use semantic HTML.
- Use real buttons for actions and real links for navigation.
- Avoid motion that makes the interface harder to use.
- Support responsive behavior across mobile and desktop.

## Recommended workflow

When using this skill, follow this order:

### 1. Define the art direction

State clearly:

- audience
- interface goal
- aesthetic direction
- memorable differentiator

### 2. Map the UI structure

Identify:

- major sections
- component hierarchy
- layout rhythm
- content priorities
- interaction hotspots

### 3. Implement production-grade code

Create:

- working markup/components
- cohesive styling
- refined states
- responsive behavior
- polished interactions

### 4. Explain the design intent

When delivering, explain:

- what aesthetic was chosen
- which typography and color decisions support it
- what motion or layout choices make it distinctive
- how usability and accessibility were preserved

## Output expectations

The final result should feel:

- intentional
- polished
- memorable
- context-aware
- production-ready
