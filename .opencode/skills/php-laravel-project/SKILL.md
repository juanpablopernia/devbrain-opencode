## Cuándo usar

- El proyecto pide framework PHP moderno.
- Se necesita API/web con productividad rápida.
- El usuario menciona Laravel explícitamente.
- Conviene usar Eloquent, migrations, colas, jobs o ecosistema Laravel.

## Enfoque

- Laravel actual.
- Convenciones idiomáticas del framework.
- Eloquent, migrations, requests, services y configuración `.env`.
- Estructura limpia, validación y separación de responsabilidades.

## Reglas

- No fuerces estilo procedural.
- No mezcles convenciones de PHP nativo con naming impropio para Laravel.
- Documenta instalación, migraciones y ejecución local.
- Crea `.env.example` sin valores reales.

## No usar por defecto

- Versiones legacy de Laravel.
- Código improvisado fuera de convenciones del framework sin razón clara.
