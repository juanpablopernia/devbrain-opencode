# Memoria del proyecto

La memoria del proyecto está basada en Markdown y optimizada para leer solo lo necesario.

## Archivos

```text
docs/brain/CONTEXT_INDEX.md
docs/brain/PROJECT_BLUEPRINT.md
docs/brain/DECISIONS.md
docs/brain/MEMORY.md
docs/brain/ERRORS_SOLVED.md
```

## Uso recomendado

- `CONTEXT_INDEX.md`: entrada rápida y mapa de lectura
- `MEMORY.md`: contexto estable del proyecto
- `DECISIONS.md`: decisiones con impacto futuro
- `ERRORS_SOLVED.md`: errores relevantes y su solución
- `PROJECT_BLUEPRINT.md`: ficha técnica viva

## Qué guardar

- Stack.
- Decisiones.
- Errores resueltos.
- Comandos.
- Rutas importantes.
- Convenciones.
- Variables `.env` esperadas sin valores.
- Preferencias del proyecto o del usuario que afecten decisiones futuras.
- Pendientes críticos de larga duración.

## Qué no guardar

- API keys.
- Passwords.
- Tokens.
- Credenciales.
- Datos sensibles innecesarios.
- Conversaciones completas.
- Logs largos o historial narrativo innecesario.

## Regla práctica

Antes de abrir toda la memoria, empezar por `CONTEXT_INDEX.md` y luego leer solo el archivo relacionado con la tarea.
